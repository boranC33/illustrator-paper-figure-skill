# Figure Contract

Use this contract before creating, rebuilding, or exporting a manuscript figure. Keep it short and concrete; it is a working checklist, not prose for the paper.

## Required Fields

- Figure number and caption target, for example `Figure 1` and the caption in `main.tex`.
- LaTeX source file and command, for example `main.tex` with `\includegraphics{figures/Figure_1.pdf}`.
- Editable master, for example `figures/Figure_1_illustrator_master.ai`.
- Published manuscript asset, for example `figures/Figure_1.pdf`.
- Review exports, for example `figures/Figure_1_export_check.pdf` and `figures/Figure_1_preview.png`.
- Component asset folder, for example `figures/fig1_illustrator_assets/`.
- Target journal or venue artwork requirements, if known.
- Nearby style reference figure, for example `figures/Figure_4.pdf`.
- Art direction contract, for example visual class, finish level, detail density, component realism, line language, material cues, and color strategy.
- Local component library decision: `none`, `create project-local library`, or `reuse project-local library`.

## Scientific Story

Write the figure story as 3--6 nodes before drawing:

```text
source observation -> abstraction -> model/calibration -> prediction -> validation
```

For each node, decide whether it should be:

- a literal component, such as a specimen, device, material region, environment, instrument, organism, or interface;
- an editable vector block, such as a process box, axis, arrow, or callout;
- a formula asset, preferably PDF/EPS from MathType or LaTeX;
- a data panel or plot that should be generated by a plotting script instead of redrawn by hand.

## Layout Decisions

- Reading direction: left-to-right, top-to-bottom, radial, or panel sequence.
- Panel labels: `(a)`, `(b)`, etc.; match caption order exactly.
- Arrow topology: straight, orthogonal, feedback loop, or branched.
- Grouping: panels, stages, input/output lanes, or mechanism zones.
- Final width: single-column, double-column, or full-page.
- Minimum readable label size at final width.
- Visual hierarchy: primary visual focus, secondary details, and label/callout priority.
- Detail strategy: which components need domain-specific geometry or material/state cues, and which should stay simplified.

## Deliverables

Prefer this deliverable set unless the journal asks otherwise:

- Editable `.ai` master, never overwritten during export-only work.
- Vector `.pdf` for LaTeX.
- `.png` preview for quick visual inspection.
- `.tif` or `.tiff` only when the journal upload system requires it.
- Asset folder containing prompts, original generated components, cleaned components, redraws, and formula assets.
- Optional project-local component library only when the user chooses it for repeated components across figures.

## Naming Pattern

Use predictable names:

```text
Figure_N_illustrator_master.ai
Figure_N_export_check.pdf
Figure_N.pdf
Figure_N_preview.png
figN_illustrator_assets/
```

Keep final LaTeX paths stable. Put experimental drafts in the asset folder or use explicit suffixes such as `_draft1`, `_redraw`, or `_manual_export`.
